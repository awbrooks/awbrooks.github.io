GitHub and Sphinx¶

This covers commands for setting up and uploading GitHub repositories, and setting up sphinx documentation.

GitHub¶

Creating a GitHub Repository¶

This assumes you have a GitHub account and the command line tools are configured on your computer.

  1. In the terminal navigate to the folder you want to upload as your repo. Set these variables.
# Project Title #
project_title="# My Project title for the README.md"

# Name the Commit (not really important) #
commit_title="first_commit"

# GitHub Address for Repo #
gh_title="git@github.com:awbrooks19/vmi_microbiome_bootcamp_II.git"
  1. Initialize the folder as a repository.
# Initialize the repo #
git init
  1. Create a readme.md and chan ge to your own custom title.
# Create readme with project title #
echo "$project_title" > README.md
  1. Add the file info to the update.
# Add all files to commit #
git add -A
  1. Commit the files to the update with your own name for the update in the quotes.
# Commit the files #
git commit -m "$commit_title"
  1. Setup the repo with the github address it will be sent to (use your own id).
# Setup repo address #
git remote add origin "$gh_title"
  1. Push everything to the online repo.
# Push the repo to github #
git push -f origin master

Sphinx¶

Updating a Sphinx Website Locally¶

sphinxb - Function to clean _build, make HTML, and open _build/html/index.html.

  • This is useful when rapidly implementing and testing changes to a sphinx webpage.
sphinxb(){
   # Function to clean _build, make HTML, and open _build/html/index.html. #
   make clean
   make html
   open _build/html/index.html
}
